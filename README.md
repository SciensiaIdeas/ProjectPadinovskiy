# Методы принятия решений в условиях неопределённости (полной, частичной, стохастической)

## Курс

**Методы принятия решений**

---

## Цель проекта

Реализация методов из учебника [1] и статьи [2] для принятия решений в условиях полной, частичной и стохастической неопределённости: критерии Вальда, Гурвица, Сэвиджа, Бернулли–Лапласа, максимального правдоподобия, попарное доминирование (Фишберн), задачи со стохастической и многокритериальной постановкой.

---

## Использование

Программу можно использовать **как библиотеку** (импорт модулей и вызов функций в своём коде) или **как консольное приложение** с двумя режимами:

- **`evaluate`** — расчёт задачи: чтение входного JSON, выполнение выбранного метода, запись результата в указанный файл.
- **`validate`** — проверка расчётных значений: сравнение результата метода с эталонным файлом (например, результаты из учебника или статей).

## Запуск (консольное приложение)

### Режим расчёта (evaluate)

```bash
dfs evaluate <модуль> <входной_файл> <файл_результата> <метод> [доп. аргументы] (-p [точность])
```
По умолчанию, точность p=2. Примеры находятся в скрипте **`test_all.sh`**.

### Режим проверки (validate)

```bash
dfs validate <модуль> <входной_файл> <эталонный_файл> <метод> [доп. аргументы]
```

Эталонные файлы лежат в каталоге **`validate/`**. Команда завершится успехом, если результат метода совпадает с эталоном (с учётом точности для вещественных полей). Примеры находятся в скрипте **`eval_economarket.sh`**.

### Автодополнение (Linux terminal)

В каталоге проекта выполните:

```bash
dfs --install-completion zsh
```

Или другой терминал (например, fish). После перезапуска терминала доступны подсказки TAB.

## Тестирование всех методов на задачах учебника

Скрипт **`test_all.sh`** позволяет протестировать сразу все задачи из учебника Подиновского [1]: он последовательно запускает `dfs validate` для заданных пар «входной файл — эталон» по всем модулям (complete, partial, stochastic) и методам. Запуск из корня проекта:

```bash
chmod +x test_all.sh
./test_all.sh
```

Успешное завершение скрипта означает, что все расчёты совпадают с эталонными результатами.

## Тестирование методов на задачах из статьи

Следующие команды позволяют решить задачу из статьи [2] (economarket.json) классическими методами полной неопределенности и методами попарного доминирования частичной неопределенности, а также её модифицированную многокритериальную версию с уровнями притязаний стохастической неопределенности и методами Вальда и Бернулли-Лапласа частичной неопределенности (economarket_modified.json):

```bash
chmod +x eval_economarket.sh
./eval_economarket.sh
```

После выполнения в директории **`results/complete`** видим результаты, которые полностью согласуются с результатами из статьи [2]. Также были получены новые результаты, которые можно наблюдать в **`results/stochastic`** и **`results/partial`**.

## Решение задач

### `algs_complete_uncertainty.py` (полная неопределённость)

- **`solution(data, method, *args)`**: обёртка над критериями полной неопределённости.  
  - `data`: `CompleteUncertaintyInput` (матрица выигрышей `M`).  
  - `method`: функция (`_pessimism`, `_optimism`, …) **или строка** с именем метода:  
    `\"pessimism\"`, `\"optimism\"`, `\"hurwich\"`, `\"savage\"`, `\"bernulli_laplace\"`, `\"maximum_likelihood\"`.  
  - Возвращает `CompleteUncertaintyResult` (веса/меры, ранги, лучшая альтернатива).
- **`general_solution(data, u)`**: считает **все** критерии сразу (Вальд, максимакс, Гурвиц, Сэвидж, Бернулли–Лаплас, максимальное правдоподобие).  
  - `u` — параметр Гурвица \([0;1]\).  
  - Возвращает `AllCompleteMethodsResult`, содержащий по одному `CompleteUncertaintyResult` на каждый критерий.

### `algs_partial_uncertainty.py` (частичная неопределённость)

- **`fishburn(input_data, ordering_type=\"weak\")`**: попарное доминирование по Фишберну.  
  - `ordering_type=\"weak\"` — нестрогое упорядочение (теорема 12.1).  
  - `ordering_type=\"strict\"` — строгое упорядочение (Кирквуд–Сарин без групп, теорема 12.2).  
  - Возвращает `CompleteUncertaintyResult` с ранжированием альтернатив.
- **`kirkwood(input_data, _groups=None)`**: доминирование при смешанном упорядочении (обобщённый критерий Кирквуда–Сарина).  
  - `_groups` — строка с Python-списком границ групп, например `\"[2, 4]\"` для `k1=2`, `k2=4`.  
  - Возвращает `CompleteUncertaintyResult`.
- **`wald_criterion(input_data, _groups)`**: критерий Вальда (максимин) на множестве вероятностей.  
  - `_groups` — строка со списком групп, например `\"[2, 4]\"`.  
  - Возвращает `WaldResult` (минимальные ожидания по вершинам и оптимальная альтернатива).
- **`bernoulli_laplace_criterion(input_data, _groups)`**: критерий Бернулли–Лапласа на том же множестве вероятностей.  
  - `_groups` — строка со списком групп.  
  - Возвращает `BernoulliLaplaceResult` (средние ожидания и оптимальная альтернатива).

### `algs_stochastic_uncertainty.py` (стохастическая неопределённость)

- **`problem_10_3(input_data)`**: задача 10.3 («Русский сыр»).  
  Строит матрицу прибыли, считает математическое ожидание и среднее полуотклонение, анализирует FSD/SSD-доминирование, возвращает `Problem10_3Result` c:
  - платёжной матрицей,  
  - списком стратегий с \((m, s^-)\),  
  - FSD/SSD-доминированием,  
  - ранжированием и рекомендованными стратегиями.
- **`multi_criteria_problem(input_data)`**: многокритериальная стохастическая задача с уровнями притязаний.  
  По `aspiration_levels` и вероятностям состояний вычисляет вероятность «успеха» каждой альтернативы и возвращает `MultiCriteriaResult` (вероятности, лучшая альтернатива и ранжирование).

## Входные и выходные данные (Pydantic-структуры из `schemas.py`)

Все форматы входных и выходных JSON заданы в модуле **`schemas.py`** с помощью Pydantic-моделей. Ниже перечислены структуры по типам задач.

### Стохастическая неопределённость

| Модель | Назначение | Поля |
|--------|------------|------|
| **Problem10_3Input** | Вход для задачи 10.3 («Русский сыр») | `demand` — список объёмов спроса; `demand_probs` — вероятности; `cost`, `price` — себестоимость и цена; `n_values` — варианты производства |
| **Problem10_3Result** | Результат `problem_10_3` | `payoff_table` — матрица прибыли; `results`, `ranked` — стратегии с \(m\), \(s^-\); `dominance_m_s`, `fsd_dominance`, `ssd_dominance`; `best_candidates`, `dominance_type`; `cdf_table` |
| **MultiCriteriaInput** | Вход многокритериальной задачи с притязаниями | `alternatives`, `states_of_nature`, `state_probabilities`; `criteria_values` — словарь «альтернатива → матрица значений по состояниям»; `aspiration_levels`; опционально `minimize_criteria`, `criteria_descriptions` |
| **MultiCriteriaResult** | Результат `multi_criteria_problem` | `results` — вероятность успеха по альтернативам; `best_variant`; `sorted_results` — список пар (альтернатива, вероятность) |

### Полная неопределённость

| Модель | Назначение | Поля |
|--------|------------|------|
| **CompleteUncertaintyInput** | Вход для всех критериев полной неопр. | `M` — матрица выигрышей (список списков чисел), размер «альтернативы × состояния» |
| **CompleteUncertaintyResult** | Результат одного критерия (Вальд, Гурвиц, …) | `measures` — вектор мер по стратегиям (от лучшей к худшей по предпочтению см. ranks); `ranks` — ранги (от лучшей к худшей, нумерация начинается с 1); `best_variant` — номер лучшей альтернативы (ranks[0]) |
| **AllCompleteMethodsResult** | Результат `general_solution` | Шесть полей: `pessimism`, `optimism`, `hurwich`, `savage`, `bernulli_laplace`, `maximum_likelihood`, каждое — `CompleteUncertaintyResult` |

### Частичная неопределённость

| Модель | Назначение | Поля |
|--------|------------|------|
| **PartialUncertaintyInput** | Вход для Fishburn, Kirkwood, Вальда, Бернулли–Лапласа | `criteria_matrix` — 2D (альтернативы × состояния) или 3D (× критерии); опционально `utility_expression` — строка функции полезности (y1, y2, …) |
| **WaldResult** | Результат `wald_criterion` | `optimal_alternative`; `min_values` — минимальные ожидания по вершинам; `n_alternatives`, `n_states`, `groups` |
| **BernoulliLaplaceResult** | Результат `bernoulli_laplace_criterion` | `optimal_alternative`; `expected_values`; `n_alternatives`, `n_states`, `groups` |

Для попарного доминирования (**fishburn**, **kirkwood**) выход — тот же **CompleteUncertaintyResult** (меры = счёт доминирований, ранги, лучшая альтернатива).
